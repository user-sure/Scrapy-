豆瓣未登录游客会限制短评、完整评分等内容展示，因此通过浏览器 Cookie 模拟登录态获取完整榜单数据，学习爬虫反爬处理、批量数据存储、管道多渠道分发等工程化能力。
核心功能实现
登录态模拟，绕过页面访问权限
使用 Edge 浏览器 Cookie-Editor 插件导出登录 Cookie，提供两种 Cookie 使用方案：原始字符串直接放入请求头、字典格式动态修改 Cookie，携带完整 UA、Referer 浏览器请求头模拟真人访问，解决游客权限限制问题。
Scrapy 全套爬虫架构开发
Items 自定义字段：定义 title、rank、score、subject 四类数据存储字段，规范数据结构；
双 Pipeline 数据持久化：
① Excel 导出管道：自动新建工作表、清理默认空白 Sheet，格式化表头，采集完成后自动保存到本地桌面 Excel 文件；
② MySQL 批量入库管道：设置缓存列表攒 100 条数据，调用executemany批量插入，大幅减少数据库 IO 开销；爬虫启动时执行 TRUNCATE 清空历史数据，捕获数据表不存在异常避免程序崩溃；
爬虫调度：配置随机请求延时 DOWNLOAD_DELAY，降低豆瓣风控拦截概率；
网页数据解析
底层基于 lxml.etree 解析网页，通过 XPath 精准提取电影标题、排名、评分、短评信息，使用item.get()容错处理缺失字段，防止爬虫抛出 KeyError 中断。
工程化配套处理
生成requirements.txt统一项目依赖清单，配合.gitignore 过滤虚拟环境、缓存、临时表格文件；
完整 Git 管理，代码上传 GitHub，支持他人一键克隆、搭建虚拟环境、一键安装依赖启动爬虫。
项目难点与解决方案
Cookie 字符串包含双引号报 Python 语法错误：使用单引号 / 三引号包裹完整 Cookie 字符串解决冲突；
单条循环插入 MySQL 效率低下：缓存批量数据批量写入，性能提升数倍；
数据表不存在执行 TRUNCATE 报错：添加异常捕获，识别 1146 表不存在错误直接跳过清空操作；
网页未登录内容截断：携带登录 Cookie 模拟账号访问，获取完整影视数据。
